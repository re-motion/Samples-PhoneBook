[COMMONS-1576] Improve performance of ClassContextCollection.GetWithInheritance

Component/s: Mixins
Issue Type: Performance
Resolution: Fixed
Status: Closed


Two problems:

  • Result of GetWithInheritance for a derived class that inherits mixins is not stored
  • GetWithInheritance for a derived class that has no mixins (and inherits none) is not stored

consider the following class hierarchy:

A <- B <- C <- D <- E

Where either none of them has mixins, or only A has mixins.

Analyzing each of these classes will lead to 15 dictionary lookups.
By storing the results, it can be done in 5-10 lookups (depending on the order).

Also consider preparing that information right at mixin configuration building time - we're iterating all types anyway.
Not sure whether it's a good idea to have an (empty) ClassContext for each type in the AppDomain, though...

Note: If you store empty ClassContexts for non-configured classes, adapt code generation to check for non-empty Mixins/CompleteInterfaces to determine whether to generate or not.
