[COMMONS-2673] BocList.js: BocList_InitializeList does not bind event handlers properly if a row is selected with single checkbox or single radiobutton

Component/s: ObjectBinding.Web
Issue Type: Bug
Resolution: Fixed
Status: Closed


Possible solution in BocList.js:

function BocList_InitializeList(bocList, selectorControlPrefix, count, selection, hasClickSensitiveRows, listMenu)
{
  var selectedRows = new BocList_SelectedRows (selection);
  if (   selectedRows.Selection != _bocList_rowSelectionUndefined
      && selectedRows.Selection != _bocList_rowSelectionDisabled)
  {
    for (var i = 0; i < count; i++)
    {
      var selectorControlID = selectorControlPrefix + i;
      var selectorControl = document.getElementById (selectorControlID);
      if (selectorControl == null)
        continue;
      var row = selectorControl.parentNode.parentNode;

      if (hasClickSensitiveRows)
        BocList_BindRowClickEventHandler(bocList, row, selectorControl, listMenu);

      if (selectorControl.checked)      
      {
        var rowBlock = new BocList_RowBlock (row, selectorControl);
        selectedRows.Rows[selectorControl.id] = rowBlock;
        selectedRows.Length++;
      }
    }
  }
  _bocList_selectedRows[bocList.id] = selectedRows;
}

Use DataManager.GetLoadedData() or LINQ instead. (Note that GetLoadedData will get the DomainObjectReference and DomainObject.State in addition to the DataContainer.)

[COMMONS-2686] WxePage now replaces the Form during the PreInit-event raised by the page isntead of during the DeterminePostBackMode phase

Component/s: Web.ExecutionEngine
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


Previous order:

  1. Page.FrameworkInitialize
  2. Page.DeterminePostBackMode, including replacement of WxeForm
  3. Page.PreInit

New order:

  1. Page.FrameworkInitialize
  2. Page.DeterminePostBackMode
  3. Page.PreInit, including replacement of WxeForm

The new order allows the dynamic setting of the page's MasterPageFile during the PreInit-phase:

override void PreInit()
{
  MasterPageFile = "..."
  base.PreInit() // replaces the WxeForm
}

[COMMONS-2575] Provide a browser-specifc css class for the body element to allow broweser-switching in stylesheets

Component/s: ObjectBinding.Web Web
Issue Type: New Feature
Resolution: Fixed
Status: Closed


Available CSS selectors:

  • body.msie6
  • body.msie7
  • body.msie
  • body.mozilla
  • body.webkit
  • body.opera

Usage:

/*apply only to MS Internet Explorer 7 */
body.msie7 div.myCssClass
{
  background-color: Red;
}

/*apply to MS Internet Explorer 8 and above */
body.msie div.myCssClass
{
  background-color: Blue;
}

/*apply to Firefox */
body.mozilla div.myCssClass
{
  background-color: Green;
}

/*apply to Chrome and Safari */
body.webkit div.myCssClass
{
  background-color: Yellow;
}

[COMMONS-2659] Refactor BusinessObjectPropertyPath to only evaluate property path once

Component/s: ObjectBinding
Issue Type: Performance
Resolution: Fixed
Status: Closed


Presently, each property is evaluated twice, once for the security check, the second time to access the value. This should be unified.
