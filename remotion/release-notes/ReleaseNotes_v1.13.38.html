[COMMONS-1966] StopWatchScope should be a class-type or provide a public default constructor

Component/s: Core
Issue Type: Improvement
Resolution: Fixed
Status: Closed


Using the StopWatchScope without considering that it is a struct-type can lead to NullReferenceException because the automatically provided default constructor leaves the internal StopWatch-instance uninitialized.

[COMMONS-1994] RelationEndPoint.SetValueFrom and DataContainer.SetPropertyValuesFrom replace internal TakeOverCommittedData methods

Component/s: Data
Issue Type: New Feature
Resolution: Fixed
Status: Closed


DataContainer.SetValuesFrom does less than TakeOverCommittedData used to do; so if you have been invoking TakeOverCommittedData via reflection until now, think carefully what you need, and call DataContainer.SetTimestamp, SetDomainObject, MarkAsChanged, Commit, etc. accordingly.
