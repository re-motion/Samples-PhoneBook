[COMMONS-2530] RdbmsProvider implementation: SelectCommandBuilder has been replaced with SingleIDLookupCommandBuilder and MultiIDLookupCommandBuilder

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


The factory methods have been removed. For a single ID lookup, use SingleIDLookupCommandBuilder. To look up multiple IDs, use MultiIDLookupCommandBuilder. To look up related IDs, use DataContainerLoaderHelper.

[COMMONS-2506] ClassDefinition.GetPropertyDefinitions(), GetRelationDefinitions(), and GetRelationEndPointDefinitions() fail when the class definition is not read-only

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


Since those methods now cache their data, they cannot be called until the class definition is in its final state.

ClassDefinitionCollection.Validate() sets all class definitions to read-only. Because the default mapping configuration is automatically validated, this breaking change only affects users manually creating class definitions and mapping configurations.

[COMMONS-1793] Allow ClientTransaction.GetObjects() to work with more than 2000 ObjectIDs

Component/s: Data
Issue Type: Improvement
Resolution: Fixed
Status: Closed


  • Integration test for ClientTransaction.GetObjects (more than 2000 IDs)
  • Integration test for TableInheritance.TestDomain.Folder.FileSystemItems with more than 2000 elements
  • Rewrite WhereClauseBuilder.SetInExpression to use an XML type parameter containing all IDs. With ADO.NET, it should be possible to set the XML parameter as a string; if not, a SqlXml value should work.

Reference statement:

SELECT s.Id,s.SomeString 
	FROM SomeString s
	WHERE s.Id in (
	select T.c.value('.','uniqueidentifier') as Id 
	from @p1.nodes('/L/I') T(c)

For XML such as:

<L>
<I>GUID1</I>
<I>GUID2</I>
</L>
