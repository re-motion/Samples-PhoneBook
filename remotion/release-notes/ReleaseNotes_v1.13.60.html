[COMMONS-2786] Discarded DomainObjects are now called "invalid"

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


The reason for this change is that new objects created in subtransactions now have discarded/invalid state in all ancestor transactions until the subtransaction is committed. Since discarded/invalid objects can now become un-discarded/valid, "discarded" is no longer the right term.

All APIs have been adjusted:

  • StateType.Invalid (was: Discarded)
  • DomainObject.IsInvalid (was: IsDiscarded)
  • DomainObjectTransactionContext.IsInvalid (was: IsDiscarded)
  • ObjectInvalidException (was: ObjectDiscardedException)
  • DataManager.InvalidObjectCount, InvalidObjectIDs, GetInvalidObjectReference (was: DiscardedObjectCount, DiscardedObjectIDs, GetDiscardedObject)
