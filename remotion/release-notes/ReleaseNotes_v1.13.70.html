[RM-3033] Support for PeristentMixins added at different points in the inheritance hierarchy

Component/s: Data
Issue Type: New Feature
Resolution: Fixed
Status: Closed


Applying a mixin to two differnt classes in the inheritance hierarchy results in an invalid mapping because the properties introduced by the mixin are added to each target class. To resolve this, the mapping has to be aware that these properties are equivalent and can be merged into a single column in the database schema.
