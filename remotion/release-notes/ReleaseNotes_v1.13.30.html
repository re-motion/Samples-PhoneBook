[COMMONS-1721] Comparer utility for MemberInfo signatures

Component/s: Core
Issue Type: New Feature
Resolution: Fixed
Status: Closed


The MemberSignatureEqualityComparer class can be used to compare the signatures of members (including generic methods) for equality. Two signatures compare equal if return/property/event type as well as any generic parameters and parameter types compare equal. Member names, parameter names, and generic parameter names are not considered.

[COMMONS-1703] Allow configuration of how assemblies are discovered by re-motion

Component/s: Core
Issue Type: New Feature
Resolution: Fixed
Status: Closed


By default, re-motion searches the bin folder for root-assemblies and loads all types from those assemblies and their referenced assemblies. Those types are then analyzed by the mixin engine, re-store, etc.

This mechanism should be made configurable.

[COMMONS-1705] Provide extensibility points for re-motion's type discovery mechanism

Component/s: Core
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


Extensibility points:

  • Configuration
    • Can specify a number of root assemblies by name or by file, using the default AssemblyFinderTypeDiscoveryService
    • Can specify a custom IRootAssemblyFinder type, using the default AssemblyFinderTypeDiscoveryService
    • Can specify a custom ITypeDiscoveryService type
    • XML => COMMONS-1703
    • Code: TypeDiscoveryConfiguration.Current
  • ITypeDiscoveryService implementation
    • Code: ContextAwareTypeDiscoveryUtility.DefaultNonDesignModeService
    • In design mode, designer's ITypeDiscoveryService is always used (via DesignerUtility.DesignerHost)

[COMMONS-1704] Allow XML configuration of how re-motion finds assemblies

Component/s: Core
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


Configuration-option:
The list of root-assemblies gets compiled using a list of AssemblyNames and Filenames

<?xml version="1.0" encoding="utf-8" ?>
<configuration>
  <configSections>
    <section name="remotion.typeDiscovery" type="Remotion.Configuration.TypeDiscovery.TypeDiscoveryConfiguration, Remotion" />
  </configSections>
  <remotion.typeDiscovery xmlns="http://www.re-motion.org/typeDiscovery/configuration" mode="CustomTypeDiscoveryService">
    <!--<customRootAssemblyFinder type="ConsoleApplication3.CustomRootAssemblyFinder, ConsoleApplication3"/>-->
    <!--<specificRootAssemblies>
      <byName>
        <include name="ConsoleApplication3" includeReferencedAssemblies="false"/>
      </byName>
      <byFile>
        <include filePattern="Castle.*.dll" includeReferencedAssemblies="true"/>
        <exclude filePattern="Castle.Core.dll"/>
        <exclude filePattern="Castle.DynamicProxy2.dll"/>
      </byFile>
    </specificRootAssemblies>-->
    <customTypeDiscoveryService type="ConsoleApplication3::CustomTypeDiscoveryService"/>
  </remotion.typeDiscovery>
</configuration>

The default of includeReferencedAssemblies is "false".
