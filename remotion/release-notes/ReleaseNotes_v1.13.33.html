[COMMONS-1600] Setting FrameworkVersion.Value programmatically does not always work

Component/s: Core
Issue Type: Bug
Resolution: Fixed
Status: Closed


The suggested way of setting the framework version to be used by VersionDependentBridge before using the ObjectFactory is the following:

Remotion.Implementation.FrameworkVersion.Value = typeof(ObjectFactory).Assembly.GetName().Version;
ObjectFactory.Create<Program>(ParamList.Empty);

However, this throws an exception because ParamList's static initializer is run before the FrameworkVersion.Value property is set.

[COMMONS-1799] Extensible enum integration with re-store

Component/s: Data
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


  • Extensible enums are stored as ID strings without type identifier. IDs can optionally include declaration spaces.
  • Properties with an extensible enum type are nullable by default and can be made non-nullable via an ExtensibleEnumAttribute.
  • The schema generator will automatically generate varchar columns with an auto-calculated column length that fits the longest ID of the respective enum type.

[COMMONS-1798] Extensible enum core functionality

Component/s: Core
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


Extensible enum types are defined by deriving from an abstract ExtensibleEnum<T> base class. Values of the enum are identified by a string that is passed to that base class and created by extension methods that target ExtensibleEnumDefinition<T>, where T is the type deriving from extensible enums. The enum class holds an instance of ExtensibleEnumDefinition<T> that allows easy discovery of the extension methods via Intellisense.

public class Color : ExtensibleEnum<Color>
{
  public Color (string identifier) : base (identifier)
  {
  }
}

public static class ColorValues
{
  public static Color Red (this ExtensibleEnumDefinition<Color> definition)
  {
    return new Color ("Red");
  }

  public static Color Green (this ExtensibleEnumDefinition<Color> definition)
  {
    return new Color ("Green");
  }
}

The ExtensibleEnumDefinition<T> class also provides services for retrieving extensible enum values by ID and for discovering the set of values belonging to a type.

New modules can extend the enum by adding new extension methods. The values are found via re-motion's standard type discovery mechanisms.

Extensible enums can optionally provide methods that are implemented or overridden by subclasses of the enum type, allowing for behavior to be attached to enum values.
