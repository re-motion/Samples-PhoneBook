[COMMONS-1904] ClientTransaction.SetRelatedObject has been removed

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


To set a related object, use ObjectEndPoint.SetRelatedObject.

[COMMONS-1899] DomainObjectCollection.Combine/UnionWith does not touch the end point if there are no items added to the collection

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


The rationale for this is that "Combine" (or "UnionWith") is really just a convenience method for if (!Contains) => Add. If nothing is added, nothing is "done" to the end point, so it isn't touched,

(If absolutely required, you can touch the end-point yourself using AssociatedEndPoint.Touch().)

[COMMONS-1897] DomainobjectCollection.IndexOf (DomainObject) now also checks whether the actual DomainObject reference is contained

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


Previously, IndexOf would only check for the ID of the given object. This was inconsistent with regards to ContainsObject and Remove, so it has been changed.
Use IndexOf (ObjectID) to check for an ID only.

[COMMONS-1889] ICollectionEndPointChangeDelegate and ICollectionChangeDelegate are removed

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


ICollectionEndPointChangeDelegate has no successor, its responsibilities have moved to EndPointDelegatingDomainObjectCollectionData.
ICollectionChangeDelegate has been replaced by the IDomainObjectCollectionData strategies.

[COMMONS-1771] DomainObjectCollection.ReplaceItems is removed

Component/s: Data
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


  • Use Clear/Add instead, or work on the underlying NonNotifyingData.
  • To intercept Commit or Rollback, use the virtual Commit/Rollback methods.

[COMMONS-1763] Compare methods are changed into extension methods, renamed to SetEquals/SequenceEqual

Component/s: Data
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


This better matches existing methods on .NET BCL classes, such as HashSet and Enumerable.

[COMMONS-1752] Constructor overload taking DomainObjectCollection is removed, constructor overload taking IEnumerable doesn't provide a makeReadOnly flag any longer

Component/s: Data
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


  • To copy the contents of a DomainObjectCollection into a new collection, either use the Clone method or the constructor taking IEnumerable<DomainObject> (use the LINQ method Cast<DomainObject>() in order to transform an IEnumerable to an IEnumerable<DomainObject> if required).
    -To get a read-only collection, use AsReadOnly.

[COMMONS-1745] Combine, GetItemsNotInCollection

Component/s: Data
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


  • Combine is renamed to UnionWith, which mimics HashSet
  • x.GetItemsNotInCollection(y) is replaced with y.GetItemsExcept(x) - this is much clearer than the current naming.
    • Note that the order of collections has changed.
    • The return value is now an IEnumerable<DomainObject>; this allows for greater flexibility. (You can easily create a new DomainObjectCollection from this enumerable if needed.)
    • The comparisons are now made based on IDs not on references. This is a change, but it better matches the other methods on DomainObjectCollection, e.g. Combine.
