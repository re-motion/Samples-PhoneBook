[COMMONS-2576] AppDomainAssemblyResolver throws a FileNotFoundException when the source app domain is shadow copied (e.g., in NUnit-Gui)

Component/s: Core
Issue Type: Bug
Resolution: Fixed
Status: Closed


The reason for this is that AppDomainAssemblyResolver's ctor uses ArgumentUtility, which lies in Remotion.Interfaces.dll. In a shadow-copied scenario, when AppDomainAssemblyResolver's ctor is called in the target AppDomain, AssemblyResolve isn't handled yet, so Remotion.Interfaces.dll cannot be found.
