[COMMONS-2173] DomainObjectCollection.OnDeleting/OnDeleted are now called in line with all other begin/end events

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


Previously, ClientTransaction.OnDeleting and ClientTransaction.OnDeleted were called in the "Perform" step of the delete operation, i.e., they occurred immediately around the change and were not in line with all other events. In fact, OnDeleting might occur after some of the changes had already been performed, and OnDeleted might occur before all changes had been performed. Due to this, OnDeleting could not be used to cancel the operation.

Now, those events are called in line with all other begin/end events occurring during a delete operation, directly after the DomainObject.Deleting event, and before the DomainObject.RelationChanging event.

Cancelling the operation from DomainObjectCollection.OnDeleting is now also safe.

[COMMONS-2166] Breaking change in the order of RelationChanging/RelationChanged/Deleted events

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


For symmetry between the operations, and because it seems to be more logical, the order of RelationChanged and Deleted events and notifications has been changed so that the end events are now raised in exactly the opposite order of the begin events.

Old order of events for relation change operation
  1. IClientTransactionListener/Extension.RelationChanging (o1)
  2. IClientTransactionListener/Extension.RelationChanging (o2)
  3. o1.OnRelationChanging/RelationChanging
  4. o2.OnRelationChanging/RelationChanging
  5. actual change is performed
  6. IClientTransactionListener/Extension.RelationChanged (o1)
  7. IClientTransactionListener/Extension.RelationChanged (o2)
  8. o1.OnRelationChanging/RelationChanged
  9. o2.OnRelationChanging/RelationChanged
New order of events for relation change operation
  1. IClientTransactionListener/Extension.RelationChanging (o1)
  2. IClientTransactionListener/Extension.RelationChanging (o2)
  3. o1.OnRelationChanging/RelationChanging
  4. o2.OnRelationChanging/RelationChanging
  5. actual change is performed
  6. o2.OnRelationChanging/RelationChanged
  7. o1.OnRelationChanging/RelationChanged
  8. IClientTransactionListener/Extension.RelationChanged (o2)
  9. IClientTransactionListener/Extension.RelationChanged (o1)
Old order of events for Delete operation
  1. orderItem.OnDeleting
  2. order.OrderItems.Removing event
  3. Relation changing events for order.OrderItems
  4. actual changes are performed
  5. order.OrderItems.Removed event
  6. Relation changed events for order.OrderItems (in same order as step 3)
  7. orderItem.OnDeleted
New order of events for Delete operation
  1. orderItem.OnDeleting
  2. order.OrderItems.Removing event
  3. Relation changing events for order.OrderItems
  4. actual changes are performed
  5. Relation changed events for order.OrderItems (in opposite order of step 3)
  6. order.OrderItems.Removed event
  7. orderItem.OnDeleted
