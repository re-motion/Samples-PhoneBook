[RM-3823] Add how to build information to spin-off projects

Component/s: Build & Tooling
Issue Type: Bug
Resolution: Fixed
Status: Closed


The "Ten Minutes Build" principle states that a developer shall be able to build a solution within ten minutes. A senior developer will understand why there is no key in the solution and that he has to generate one to build the solution, a junior developer probably will fail.

For the re-motion project, there is this kind of information. This information should be also provided with the spin-off projects in codeplex. In the attachment, there is the shortened version for the re-mix project of the original re-motion how to build.txt. Also the Generate-Snk.cmd should be added.
