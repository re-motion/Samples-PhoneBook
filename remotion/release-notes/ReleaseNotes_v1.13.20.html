[COMMONS-1523] Bi-directional relation property incorrectly updated to database

Component/s: Data
Issue Type: Bug
Resolution: Fixed
Status: Closed


Problem
Given:

  • a parent and child object having a 1:N relation (1 parent - many children).
  • parent1 is related to child1 and child2

error producing sequence:

  • in a SubTransaction of a RootClientTransaction
  • child2 is disconnected from parent1 and deleted
  • new parent2 object is created
  • child1 is disconnected from parent1 and connected to parent2
  • parent1 is deleted
  • both transactions are committed

produces error

  • child1 is saved to database with child1.Parent == null ( should be child1.Parent == parent2)

Demonstration
In Acta Nova branch of remotion the unit test MultiLevelTransactionTest.ChangeParent() demonstrates the error.

Severity
This error curently produces an unfixable erroneous behaviour in Acta Nova workflow - customer waits for fix

Possible cause
When commiting the SubClientTransaction to its parent transaction the delete of parent1 causes an overwrite of child1.parent with a null value, which previously has been correctly set to parent2. Somehow parent1 still thinks it is connected to child1.
The erroneous behaviour occurs somewhere in RelationEndPoint processing.
