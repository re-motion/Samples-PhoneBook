[COMMONS-2094] ClientTransaction.LoadRelatedObjects return type is now DomainObject[] and does not register the end point any longer

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


The method has been changed in order to support lazy collection loading. It should not be called by client code, use RelationEndPointMap.GetRelationEndPointWithLazyLoad() instead.

[COMMONS-2147] ClientTransaction.EnsureDataAvailable for relation end points, DomainObjectCollection.IsDataAvailable, DomainObjectCollection.EnsureDataAvailable

Component/s: Data
Issue Type: New Feature
Resolution: Fixed
Status: Closed


These APIs can be used to detect whether and ensure that a (collection-valued) relation property's data is available in the ClientTransaction. They are valuable when used together with DomainObjectUnloader.UnloadCollectionEndPoint().

[COMMONS-2064] DomainObjectUnloader.UnloadData

Component/s: Data
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


Removes an object's DataContainer from the ClientTransaction. In addition, all relation end points that were created when the object was loaded, are also removed. If the object is part of a collection-valued relation end point, that end-point is also transitioned into unloaded state.

[COMMONS-2060] DomainObjectUnloader.UnloadCollectionEndPoint

Component/s: Data
Issue Type: Sub-task
Resolution: Fixed
Status: Closed


Transfers a collection-valued end point into unloaded state, the DomainObjectCollection representing the relation end point becomes a lazily-loaded proxy.
