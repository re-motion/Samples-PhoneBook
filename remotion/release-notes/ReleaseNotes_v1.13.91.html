Details

[RM-3681] ClientTransaction.Load... methods have been obsoleted

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


The ClientTransaction.Load methods have been obsoleted. These methods were never meant to be called by the user. To intercept objects being loaded, provide a custom IObjectLoader to the ClientTransaction when it is created.

[RM-3675] ClientTransaction.GetObject (and accessing relation of invalid objects) now throw an ObjectInvalidException for invalid objects

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


Previously, an ObjectNotFoundException would be thrown for reasons of compatibility with previous versions.
However, this exception is misleading; ObjectInvalidException is really more suited to this error.

[RM-2990] Use full column lists instead of stars for entities in re-store LINQ queries

Component/s: Data.Linq for re-store
Issue Type: New Feature
Resolution: Fixed
Status: Closed


This is required for several features already supported for re-linq, most notably for grouping by entities (group by order.Customer) and for complex nested expressions (select new { O1 = orderItem1.Order, O2 = orderItem2.Order }).
