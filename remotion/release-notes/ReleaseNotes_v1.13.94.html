[RM-3725] Mixin.This and Mixin.Base have been renamed to Target and Next

Component/s: Mixins
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


"This" was ambiguous, since the mixin already had a (different) "this" reference. This became visible especially with the mixin-based composition pattern. "Target" is much clearer.

Similarly, "Next" better expresses the fact that this property represents the next implementation of an operation in an override chain (which can contain other mixins in addition to "base" methods).
