[COMMONS-1862] TypeProvider claims that properties of type Enum or IExtensibleEnum are supported

Component/s: Data
Issue Type: Bug
Resolution: Fixed
Status: Closed


Properties of type Enum or IExtensibleEnum seem to work at first, but when loading or saving an object, a persistence-related exception is thrown.
Instead of this, there should be an exception at mapping construction time.

[COMMONS-1866] API change of ClientTransaction.GetObjects and TryGetObjects

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


The return type of ClientTransaction.GetObjects and TryGetObjects has changed from ObjectList<T> to T[]. Clients requiring dictionary-based lookup should use the Enumerable.ToDictionary method to transform it into a dictionary.

The arrays returned by TryGetObjects and GetObjects (..., false) will now contain null values for IDs that couldn't be loaded. This results in the returned array directly matching the array of IDs passed into the method, but clients must be able to deal with null values.

[COMMONS-1861] TypeProvider.SupportedTypes is removed

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


The reason for this is that the dictionary-based approach cannot be used to express supported base classes or interfaces.

Instead, use IsTypeSupported to check whether a type is supported, or use one of the other TypeProvider APIs to add new supported types.

Use GetNonNotifyingData instead.

[COMMONS-1844] Configurable extensible enum value order

Component/s: Core
Issue Type: New Feature
Resolution: Fixed
Status: Closed


Attach ExtensibleEnumPositionAttribute to the extension method defining a value to assign it a floating point positional key. That key determines the position of that value in the list of all values for the extensible enum type.

Values that do not have the attribute applied default to 0.0. Values with equivalent keys are sorted alphabetically (by full ID).
