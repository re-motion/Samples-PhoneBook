[COMMONS-1913] ClientTransaction events, extensions, and listeners currently getting read-only DomainObjectCollections now receive ReadOnlyCollection instances

Component/s: Data
Issue Type: Breaking Change
Resolution: Fixed
Status: Closed


This is more performant and more clearly communicates the intention.

ReadOnlyCollection<DomainObject> provides most of the functionality a read-only DomainObjectCollection does. In case access by ID is needed, a Dictionary can be created by the client.
